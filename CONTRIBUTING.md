# 贡献指南

感谢你对 Duckov Custom Model 项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了一个 Bug，请：

1. 在 [Issues](https://github.com/Duckov-Custom-Model/DuckovCustomModel/issues) 中搜索是否已有相关报告
2. 如果没有，请创建一个新的 Issue，使用 Bug Report 模板
3. 提供尽可能详细的信息，包括：
   - 问题描述
   - 复现步骤
   - 预期行为和实际行为
   - 环境信息（游戏版本、模组版本、操作系统等）
   - 相关日志或截图

### 提出新功能

如果你有功能建议，请：

1. 在 [Issues](https://github.com/Duckov-Custom-Model/DuckovCustomModel/issues) 中搜索是否已有相关建议
2. 如果没有，请创建一个新的 Issue，使用 Feature Request 模板
3. 详细描述功能需求和使用场景

### 提交代码

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 代码规范

- 遵循项目现有的代码风格
- 确保代码可以正常编译
- 添加必要的注释（如果现有代码有注释，请遵循相同的注释风格）
- 确保没有引入新的警告或错误

### 开发环境设置

1. 克隆仓库
2. 使用支持 .NET Standard 2.1 的 IDE（如 Visual Studio、Rider 等）打开解决方案
3. 还原 NuGet 包
4. 开始开发

## 行为准则

请遵守我们的 [行为准则](CODE_OF_CONDUCT.md)。

## 联系方式

如有问题，可以通过以下方式联系：

- 创建 Issue
- 在 Discussions 中讨论
- 发送邮件至：duckov-custom-model@safety-email.ritsukage.com

感谢你的贡献！
